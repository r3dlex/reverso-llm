# SDLC Path: Brownfield Adoption

## Repository

`reverso` - standalone python ASGI/asyncio LLM gateway (depth 0).

## Path chosen

**Brownfield adoption** - the repo already has a partial v3 scaffold (`.ai/rules/`,
`.ai/skills/`, `.ai/system-prompts/`, `.ai/drift/`, `.memory/`, `docs/architecture/`,
`docs/specifications/`, `docs/learning/`, ADR-0001 through ADR-0012) generated by
earlier `init-ai-repo` runs and project work. The v3 upgrade adds the missing
governance layers additively without modifying or deleting existing content.

## Rationale

- Existing `.ai/matrix.json` already declares `topology_type: standalone`,
  `current_depth: 0`, `sync_strategy: physical-copy` - correct for this repo.
- `.memory/human-override/` is populated; terminal priority is maintained.
- CI (GitHub Actions) is already configured (`ci.yml`, `ci-prek.yml`,
  `release.yml`); no hosted branch-policy mutations needed (protected `main`,
  PR-only delivery).
- The CI gate is `uvx prek run --all-files` plus `scripts/validate-rules.sh`,
  `scripts/archgate.sh`, and `uv run pytest`; this upgrade is docs/governance
  only and does not change any of those gates.
- The repo is not a skills catalog - the skill-modernization branch is
  inapplicable.
- Cascade is inapplicable (standalone topology has no managed child repos).

## Key decisions

| Decision | Choice | Reason |
| --- | --- | --- |
| Topology | standalone, depth 0 | Single-repo python gateway; no umbrella. |
| Sync strategy | physical-copy | Canonical per `modules/sync.md`; symlinks/submodules rejected. |
| Tracker | GitHub Issues (hosted) | `r3dlex/reverso-llm` is on GitHub; PR-only delivery on protected `main`. |
| Cascade | no-op | Standalone topology; no managed child repos. |
| Skill catalog | excluded | This repo is not a skill catalog. |
| Example evalsets | not shipped | No graded AI surface requiring evaluation at upgrade time. |
